Form

    폼컨트롤을 하나 만들었다고 할 때
    전송버튼을 누르면 액션에 연결된 값으로 전송이 된다.
    서버사이드 프로그래머가 작성을 해서 이 부분을 처리하게 된다.

    전송방식으로는 get, post 방식이 있다.
    
    get 은 필드명과 값이 출력되므로 보안에 취약하다.
    대부분 post 방식을 사용한다고 보면 된다. 
    (get 을 사용하는 경우는 검색 필드부분을 만들 때 사용할 수 있다)

    fieldset 은 필수라기 보다는 input 을 여러개 만들 때 각각의 서식을 묶어주기 위해 사용한다.
    fieldset 으로 묶어주면 선이 나오게 되는데, 디자인을 위해서 CSS로 fieldset 선을 가리고, legend도 가려서 작업한다.

    요즘은 굳이 취소버튼을 만들지 않는 추세이다. 왜냐하면 버튼을 통하지 않더라도 창을 닫을 수 있기 때문인데,
    만약 취소버튼을 만든다면 디자인을 전혀 다르게 해서 만드는데, 실수로라도 누를 수 있는 가능성을
    막기 위해서이다.
    
    

    
Form을 구상할 때 유의할 점(구조설계)

    form 을 짤 때는 action, 즉 전송방식을 만들고,
    form 안에는 서식을 묶어줄 수 있는 fieldset으로 그룹을 만들고,
    그 그룹의 이름은 legend 로 뽑아준다.
    
    또한 각 input 항목은 블록 레벨 요소를 사용하여 감싸주는 게 좋다.
    각 input 항목을 만들 때는 요소제목을 레이블로 감싸고,
    해당 label의 for 값과 input 요소(컨트롤)의 id 값을 매치시킨다.

    그리고 마지막으로 전송버튼을 만든다. 

    input 태그 안에 실제 입력하는 값이 value 값이 된다.
    미리 입력할 필요는 없으나 예시를 보여주기 위해서 사용할 수도 있는데
    value 를 사용하면 값을 입력하기 위해 지워야하므로 미리 입력할 필요는 없다.
    placeholder 가 좋은 대안이 될 수도 있다.
    

    
input 요소의 속성별 주의사항


    cols, row 는 각각 한 글자, 한 줄의 단위를 갖는다.
    이들 속성을 뺀다면 오류가 생길 수 있으므로 디자인은 CSS로 하더라도 HTML 상에서는 빼놓지 말아야 한다.

    select 는 value 값이 전송되므로
    영어, 숫자 등 어떤 값과도 매치할 수 있다. 그러므로 개발자의 영역!

    라디오에서는 name 값이 같아야 한다.

    Tip. 
    label 이 input 태그를 감싸고 있으면 id와 for를 사용하여 연결하지 않아도 
    연결이 돤다. 보통 체크박스와 라디오를 사용할 때 가장 많이 사용한다! 
    그렇지만 디자인을 하기 위해서 보통 label과 input 태그를 분리해서 사용한다.
    
    button 태그와 input 으로 만든 버튼은 기능은 같지만
    디자인적으로는 버튼태그가 유리할 때가 많다.
legend 를 사용하여 어떤 내용이 있는지 알려준다.
