input

        HTML5 로 업그레이드 되면서 input 타입들이 다양해졌다.

        브라우저 마다 표현이 다르다. 
        지원이 되지 않는 속성이 있으므로 주의한다.    
    
        OS 및 브라우저별로 공통적으로 제공하는 것은
        가져다 쓰고 그렇지 않은 것들은 제작을 한다. 

        휴대폰에서는 @, .(dot)등을 쓰기 편하게 가상 키보드로 제공한다.
        그럼 사용자 입장에서 고려할 수 있게 된다. 오타를 줄일 수도 있다. 

        가장 많이 사용되는 것은 
        date, email, number, search, tel, url 이 있다. 
        이외에는 지원을 안 하거나, UI가 이쁘지 않아서 잘 사용되지 않는다.

  • color picker, 18년 1월기준 아이폰 지원X

  • date, 달력 디자인은 css로 제어할 수 없다. OS별로 달라서 잘 쓰이지 않기도 한다. UI 통일을 원하는 경우 쓰이지 않음

  • date time, 18년 1월 기준 아직 지원을 하지 않음

  • datetime-local, 크롬, Andro, ios 가능

  • month, 기본적으로 제공하는 것을 사용하는 게 편하다. 크롬, Andro, ios 가능

  • week, ios는 지원X

  • time, 크롬, Andro, ios 가능

  • email, ios는 지원X

  • number, 숫자키패드를 생성(사용자 친화적)

  • range, 잘 쓰지는 않음..!

  • search, 검색영역을 만들 때 쓴다. 모바일에서는 돋보기 or return 버튼을 제공한다.

  • tel, 모바일에서는 숫자 키보드를 제공한다 별, 샵, 플러스 표시가 있어서 특수문자를 쓸 수 있도록 제공한다. ios에서 tel은 숫자키패드만 나온다.

  • url, 도메인을 입력할 때 입력을 돕는다. 키패드가 www. 과 .com 이 제공된다.
