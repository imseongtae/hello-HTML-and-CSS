EMS - 이메일 이벤트 페이지

이메일 이벤트 페이지를 만들 때에는 다양한 기기와 브라우저 및 어플리케이션 환경을 고려하여 최대한 높은 호환성을 가질 수 있도록 태그를 작성한다.

dtd(Document Type Definition, DTD | 마크업 언어에서 문서 형식을 정의하는 것)는 버젼을 낮춰서 쿽스 모드(Quirks mode)로 작성한다.
높은 호환성을 고려하기 위해서 html5가 아닌 보다 호환성이 높은 독타입을 선언한다.
낮은 버젼의 독타입을 선언하거나 독타입을 쓰지 않는다.
높은 호한성을 위해서 고전적으로 태그를 작성한다.
지금은 잘 사용하지 않는 bgcolor나 테이블 내 border:0, inline-style이 아닌 방법으로 스타일 등을 주는 방법(인라인 태그에 그냥 width를 사용) 등을 사용한다.
레이아웃은 table을 사용한다
고전적인 방법인 table 사용을 통해서 작성한다.그래야 호환성이 높다.
배경색은 table 이나 td 에 넣어준다.
tr에 넣는 방법은 호한성을 떨어뜨린다는 것을 생각한다.
인라인 태그를 사용한다
body 태그 안의 내용만 보여줄 수 있으므로 인라인 태그를 사용해 스타일을 작성한다.
이미지를 넣을 때 style 속성애 vertical-align:top; border:0; 을 꼭 넣어준다.
간격이 발생하는 문제를 없애기 위해서
이미지를 넣을 때 절대경로를 사용한다.
상대경로를 통해서는 보여줄 수 없으므로 어딘가에 올려져 있는 그림의 주소를 받아와서 그 그림의 경로를 사용해야만 한다.
이메일 이벤트 페이지를 만들 때에는 호환성을 고려해서 옛날 방식으로 마크업을 한다. 이미지는 버티칼 얼라인 탑, 보더 0을 꼭 넣어준다. 배경색은 table 이나 td에 넣어준다 레이아웃을 짤 때는 table을 사용한다. 바디안에 있는 코드만 전송되는 점을 유의해서 스타일을 인라인으로 작성한다. 그림은 절대경로로 작성 dtd는 버젼을 낮춰서 쿽스 모드로 작성한다
